## Backend: `ratatui-termina`

Ratatui now has a backend for [Termina](https://github.com/helix-editor/termina)!

:::note[About Termina]

[Termina](https://github.com/helix-editor/termina) is a cross-platform VT manipulation library
created for the [Helix editor](https://helix-editor.com/). It combines ideas from Crossterm and
TermWiz while exposing a lower-level API for terminal escape codes, with the goal of making newer
terminal capabilities easier for applications to detect and handle.

:::

The new `ratatui-termina` crate is available through the `termina` feature and is re-exported from
the main `ratatui` crate:

```toml
[dependencies]
ratatui = { version = "0.30.2", features = ["termina"] }
```

```rust
use ratatui::backend::TerminaBackend;
use ratatui::termina::{PlatformTerminal, Terminal as _};
use ratatui::Terminal;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let mut output = PlatformTerminal::new()?;
    output.enter_raw_mode()?;

    let backend = TerminaBackend::new(output);
    let mut terminal = Terminal::new(backend)?;

    terminal.draw(|frame| {
        frame.render_widget("Hello from Termina!", frame.area());
    })?;

    Ok(())
}
```

This gives Ratatui another backend option while keeping the same `Backend` abstraction that widgets
and applications already use.

Let us know if you have any feedback or run into any issues with the new backend! рҹҗҖ

## Widgets: `Send` and `Sync` again

Ratatui 0.30.1 introduced [block shadows](https://ratatui.rs/highlights/v0301/#block-shadows), but
the internal storage for custom shadow effects made some widgets stop implementing `Send`. This
showed up as compile errors in downstream apps as follows:

```text
error[E0277]: `(dyn ratatui::widgets::CellEffect + 'static)` cannot be shared between threads safely
```

Ratatui 0.30.2 fixes that regression! The widgets are `Send` and `Sync` again when their contents
allow it, so existing threaded app architectures should compile as before.

:::note[Details]

For custom shadow effects, the `CellEffect` trait now carries the required auto-trait bounds:
`Send`, `Sync`, `UnwindSafe`, and `RefUnwindSafe`. That makes the requirement explicit and prevents
custom effects from accidentally removing those guarantees from widgets that contain a `Block`.

See the [pull request](https://github.com/ratatui/ratatui/pull/2584) for more details.

:::

## Buffer: Wide Cell Cleanup

Ratatui now correctly updates cells that are uncovered when a wide character is replaced by a
normal-width cell.

Here is an example of the issue that this fix addresses:

```rust
// frame 1
Span::raw("дҪ еҘҪпјҢдё–з•ҢпјҒ").bg(Color::Blue);

// frame 2
Span::raw("Hello");
```

First frame (blue background):

